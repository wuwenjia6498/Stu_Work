# 任务目标
在现有的海报生成器工具中，新增两个基于 AI 的功能：
1. **手写稿 OCR 识别**：用户上传手写图片后，调用 AI 提取文字，用户在弹窗中审核/修改后填入正文。
2. **AI 生成评语**：根据当前表单中的“习作正文”和“学员信息（包含年级）”，一键生成老师评语。

# 技术与接口规范
1. **AI 模型配置**：
   - 必须硬编码模型名称：`gemini-2.5-pro`
   - 必须硬编码 Base URL：`https://api.aihubmix.com/v1` （请使用标准的 OpenAI 兼容格式进行请求）
   - **不要**将 Base URL 或 Model name 放进 `.env.local`。
   - 仅从 `.env.local` 读取 API Key，环境变量命名为 `AIHUBMIX_API_KEY`。
2. **后端路由**：
   - 请在 `app/api/ai/route.ts` (或合适的 API 目录) 创建一个统一的后端接口来转发 AI 请求，绝对不能在前端暴露 API Key。
   - 接口需支持处理多模态（图片+文本）和纯文本请求。

# 需求拆解与 UI 交互

## 功能 1：手写稿 OCR 识别
- **UI 位置**：在左侧“习作正文”文本框上方或右侧，增加一个“上传手写稿识别”的次级按钮 (可用 shadcn 的 `Button` variant="outline" 加个小图标)。
- **交互流程**：
  1. 用户点击按钮，选择图片文件。
  2. 按钮进入 loading 状态，调用后端 API 发送图片。
  3. AI 提示词要求：“请识别并提取这张图片中的手写文字，忽略涂抹修改的部分。只输出识别到的纯文本内容，不要有任何多余的解释。”
  4. 接口返回结果后，弹出一个弹窗 (shadcn `Dialog`)。
  5. 弹窗内包含一个 `Textarea` 显示识别出的文字，允许用户二次编辑修改。
  6. 弹窗底部有“确认填入”按钮，点击后将文字覆盖或追加到原有的“习作正文”状态 (`posterData.content`) 中，并关闭弹窗。

## 功能 2：AI 一键生成评语
- **UI 位置**：在左侧“老师评语”输入框附近，增加一个“AI 智能生成”按钮。
- **交互流程**：
  1. 点击按钮前，前端校验 `posterData.content` (正文) 是否为空，为空则 Toast 提示需先输入正文。
  2. 点击后，按钮进入 loading 状态。
  3. 调用后端 API，传入当前的正文内容和学员信息（尤其是年级）。
  4. AI 提示词要求：“你是一位有十几年经验的资深儿童阅读指导老师。请根据以下[年级]学生的习作内容，写一段50-80字的老师评语。语气要鼓励、专业、有针对性。只输出评语正文，不要输出其他废话。\n年级/学员信息：{学员信息}\n习作内容：{习作正文}”
  5. 接口返回后，将生成的评语直接填入 `posterData.teacherComment` 中，用户可继续在原文本框内手动微调。
